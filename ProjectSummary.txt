# SepticMonitor IoT System - Project Summary

## 1. Project Overview
A dual-purpose IoT system designed to monitor a septic tank liquid level and track pump activity. The system uses an Arduino Nano ESP32 at the tank (solar-powered) and a Shelly Plus smart plug at the house to monitor pump power. Data is centralized in Firebase and displayed on a custom Android app.

## 2. Hardware Architecture
*   **Controller:** Arduino Nano ESP32 (S3).
*   **Sensor:** Dioche A02YYUW Waterproof Ultrasonic Sensor (UART version).
    *   **Logic:** Triggered via 0x55 byte; 9600 baud.
*   **Pump Monitor:** Shelly Plus (120V AC) at IP 192.168.86.43.
*   **Power:** Solar Panel -> 12V Battery -> 12V-to-5V Step-down.

## 3. Firmware (Arduino/ESP32) Status
*   **Primary File:** `esp32/SepticMonitor/SepticMonitor.ino`
*   **Features:**
    *   **Smart Intervals:** Checks pump wattage every 5s; checks tank distance every 30s (Standard) or 15s (Test Mode).
    *   **Remote Calibration:** Fetches `tank_empty` and `tank_full` from Firebase `/settings` node.
    *   **Pump Analytics:** Calculates exact "Run Duration" (e.g. "2 min 15 sec") and logs unique start times to `/pump_log`.
    *   **Daily Heartbeat:** Counter resets to 0 at midnight (NTP synced) to show "Readings Taken Today".
    *   **Mode Control:** Automatically quits "Test Mode" and returns to 30s interval when requested by the app.
    *   **Watchdog:** Auto-reconnects to Wi-Fi; reports Shelly "Offline" status if connection to the plug fails.

## 4. Android App Status
*   **Dashboard:** Real-time progress bar for tank level, dynamic color-coded system status (Green/Yellow/Red).
*   **Calibration UI:** Dedicated Settings screen to update tank dimensions in the cloud.
*   **History & Logging:** Displays the last 5 unique level changes and the most recent pump run time/duration.
*   **Daily Report:**
    *   Automated 12:00 PM notification via WorkManager.
    *   Generates a full system report email to mackeyrj@gmail.com.
    *   "Wait logic" ensures cloud data is live before launching the report.
*   **Command Control:** "Refresh Reading" button triggers an immediate sensor check and disables high-frequency Test Mode.

## 5. Cloud (Firebase) Structure
*   **URL:** https://septicmonitor-61662-default-rtdb.firebaseio.com
*   **`/status`**: Real-time updates for levels, pump state, heartbeat, and duration.
*   **`/settings`**: Remote calibration values for the sensor.
*   **`/pump_log`**: Historical archive of all pump activity using unique Push IDs.

## 6. Current Milestone
The system is now fully feature-complete for field installation. It has passed stability testing for remote calibration, daily heartbeat resets, and pump duration tracking. The transition from "Test Mode" to "Standard Mode" via the app is fully operational.

## 7. Future Goals
*   **Advanced Analytics:** Counting and displaying the total number of pump runs per week inside the dashboard.
*   **Power Optimization:** Implementing "Deep Sleep" between readings if solar battery levels drop below a certain threshold.
